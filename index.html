Text Mining with D3.js

How it works:

  1. The page first scrapes Yelp.com for a list of restaurant reviews; in this case, for the Bottega Louie restaurant downtown. I chose Yelp mostly because it is well-known and it has a lot of public reviews. Proxied web scraping was necessary because Yelp's developer API is, unfortunately, very stingy about releasing JSON or XML for those reviews - it allows one review per AJAX call, which would be pretty cumbersome to work with.
  2. After collecting the reviews, the page uses the Aylien Text Analysis API to find the most distinct subject matter concepts in the reviews, and pulls out the top five. Please be patient with this step. The API can sometimes take over a minute to process the reviews.
  3. The page aggregates the reviews by concept. It then uses a different part of the same API for sentiment analysis, meaning, the overall positive or negative sensibility of all reviews containing each concept. I chose aggregate assessment, instead of individually assessing each review, because high-level concepts wouldn't be (and ought not to be) influenced very much by any single review.
  4. The results are ordered on the y-axis by polarity. Positive concepts are blue, neutral concepts are green, and negative concepts are red. The font size of the word reflects the confidence that each concept was correctly identified relative to the concept with the highest reported confidence (larger = higher confidence). Word opacity reflects the confidence of the polarity assessment (more opaque = higher confidence).
  5. Clicking on each word loads a tooltip with the raw numbers. Clicking inside the tooltip closes it.
